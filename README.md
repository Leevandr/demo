# Магазин обуви

Проект сделан для демонстрационного экзамена по специальности
09.02.07 "Информационные системы и программирование".

Тема проекта: приложение для магазина обуви.

В приложении можно войти под разными ролями, посмотреть список товаров,
искать и фильтровать товары, работать с заказами, добавлять и редактировать
данные через базу данных MySQL.

## Что использовалось

- Python 3.13
- PyQt6 для оконного приложения
- MySQL для базы данных
- PyMySQL 1.2.0 для подключения Python к MySQL
- PyInstaller для сборки exe-файла

PyMySQL - это библиотека-драйвер для работы с MySQL из Python. Она написана
на Python и позволяет подключаться к базе данных, выполнять SQL-запросы,
получать строки из таблиц и обрабатывать ошибки MySQL.

В проекте PyMySQL используется в файле `src/db.py`.

## Как запустить проект

1. Установить MySQL или запустить локальный сервер MySQL.
2. Создать базу данных из файла `sql/shoes.sql`.
3. Проверить настройки подключения в файле `src/db.py`.

Сейчас в проекте указаны такие настройки:

```python
database="shoes"
user="root"
password=""
port=3308
host="localhost"
```

Если на компьютере другой порт, пароль или пользователь, их нужно поменять в
`src/db.py`.

4. Установить зависимости:

```bash
pip install -r requirements.txt
```

5. Запустить приложение:

```bash
python src/main.py
```

Также в проекте есть готовый exe-файл:

```text
dist/DemoExam.exe
```

## Роли в системе

Гость:

- может смотреть список товаров;
- не может искать, фильтровать и сортировать товары;
- не может работать с заказами;
- не может добавлять, редактировать и удалять данные.

Клиент:

- может войти по логину и паролю;
- может смотреть список товаров;
- не может работать с заказами;
- не может добавлять, редактировать и удалять данные.

Менеджер:

- может смотреть список товаров;
- может использовать поиск, фильтр по поставщику и сортировку по количеству;
- может смотреть заказы;
- не может добавлять, редактировать и удалять товары и заказы.

Администратор:

- может смотреть товары;
- может искать, фильтровать и сортировать товары;
- может добавлять, редактировать и удалять товары;
- может смотреть заказы;
- может добавлять, редактировать и удалять заказы.

## Что реализовано

- окно авторизации;
- гостевой вход;
- скрытие пароля при вводе;
- отображение ФИО пользователя в главном окне;
- выход из учетной записи обратно на окно входа;
- разграничение прав по ролям;
- вывод товаров из базы данных;
- вывод изображения товара;
- картинка-заглушка, если изображения нет;
- поиск товаров по текстовым полям;
- фильтрация товаров по поставщику;
- сортировка товаров по количеству на складе;
- совместная работа поиска, фильтрации и сортировки;
- добавление товара;
- редактирование товара;
- удаление товара;
- запрет удаления товара, если он используется в заказе;
- вывод заказов;
- добавление заказа;
- редактирование заказа;
- удаление заказа;
- обновление списков после добавления, редактирования и удаления;
- проверка даты доставки заказа;
- сохранение изображения товара в папку приложения;
- ограничение изображения товара до 300x200;
- удаление старого изображения при замене, если оно больше не используется.

## Цветовая подсветка товаров

Если скидка товара больше 15%, карточка товара подсвечивается зеленым цветом
`#2E8B57`.

Если товара нет на складе, карточка товара подсвечивается голубым цветом.

Если у товара есть скидка, основная цена показывается красным цветом и
зачеркивается. Рядом выводится итоговая цена черным цветом.

## База данных

SQL-скрипт базы данных находится в файле:

```text
sql/shoes.sql
```

В базе есть таблицы:

- `users` - пользователи;
- `roles` - роли пользователей;
- `products` - товары;
- `categories` - категории товаров;
- `manufactures` - производители;
- `suppilers` - поставщики;
- `units` - единицы измерения;
- `orders` - заказы;
- `order_statuses` - статусы заказов;
- `pickup_points` - пункты выдачи.

В таблицах используются первичные и внешние ключи.
